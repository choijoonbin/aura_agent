### 프로젝트 명

Aura Agentic AI 기반 실시간 재무 리스크 탐지 및 지능형 자율 워크벤치

(실제 구현: **Aura Agent AI PoC** — FastAPI + Streamlit + LangGraph 단일 코드베이스)

---

### 배경 및 문제 정의

* **현재 상황 (As-Is):** 수기 중심의 재무 모니터링 및 규칙 기반(Rule-based) 사후 통제 시스템 운영

* **Pain Point:** 방대한 SAP 전표 데이터 검토에 따른 리드타임 과다 발생

  * **비용:** 단순 반복적인 미결제 항목(Open Items) 추적에 따른 인적 자원 낭비  
  * **품질:** 담당자 숙련도에 의존한 오류 탐지로 인해 부정 전표나 중복 송장 누락 리스크 상존

* **기회 요인:** 에이전트형 AI가 사내 규정집을 RAG로 활용하고, 판단 근거(Reasoning)를 제시하며, HITL(Human-in-the-Loop)로 검증 가능한 자율 조치 환경을 구축할 수 있음. 본 PoC는 **LangGraph 기반 자율형 에이전트**가 전표를 분석·규정 위반 여부를 판단하고, 구조화된 이벤트 스트리밍(SSE)과 HITL 체크포인트를 통해 설명 가능한 감사 결과를 제공하는 형태로 구현되어 있다.

---

### 목표 사용자 (Target User)

* **재무/회계 실무자:** AI가 분류해준 고위험 케이스만 집중 검토하여 업무 효율을 극대화하려는 담당자  
* **CFO 및 관리자:** AI의 추론 근거를 바탕으로 데이터 기반의 신속한 승인 및 의사결정을 원하는 관리자  
* **감사(Audit) 팀:** AI의 분석 이력과 데이터 추적(Lineage)을 통해 투명한 감사 증적을 확보하려는 관리자

---

### 프로젝트 목표

* **정성적 목표**

  * **목표 1 (자율 판단):** 단순 알림을 넘어 AI가 규정 위반 여부를 스스로 판단하고 조치 대안을 제시하는 에이전틱 환경 구현.  
    → **현재:** LangGraph 기반 2단계(Phase 0 스크리닝, Phase 1 심층 분석) 오케스트레이션, planner(조사 계획·도구 선택) 기반 도구(tool) 선택 및 순차/병렬 실행, critic(과잉 주장·반례 검토)·verify(게이트·HITL 판단)·reporter(최종 설명·요약)·finalizer(최종 확정)를 통한 근거 기반 판단 및 요약 생성.

  * **목표 2 (설명 가능한 AI):** "왜 위반인가?"에 대해 학습된 규정 조항을 근거로 제시하는 **Reasoning Chain** 구축.  
    → **현재:** RAG 규정집(policy_rulebook_probe), 전표·증빙 수집(document_evidence_probe), 휴일/예산/가맹점 전용 probe 및 도구 결과 기반 점수·citation·판단요약 생성. SSE로 NODE_START/END, TOOL_CALL/TOOL_RESULT, HITL_REQUESTED 등 구조화 이벤트 스트리밍.

  * **목표 3 (신뢰 기반 승인):** 사람이 최종 승인(HITL)과 AI의 자율 판단이 결합된 하이브리드 거버넌스.  
    → **현재:** verify(게이트·HITL 판단) 노드에서 HITL 승격 판단, hitl_pause에서 interrupt, 구조화된 hitl_request(reasons, questions, handoff). 응답 제출 후 동일 run으로 resume → hitl_validate → reporter(최종 설명·요약) → finalizer(최종 확정) 재실행.

* **정량적 목표 (KPI)**

| **성과 지표**       | **목표 수치** | **측정 방법**                   | **현재 수준**       |
| ------------------ | ------------- | ------------------------------- | ------------------- |
| 이상 전표 탐지 정확도 | 95% 이상      | AI 탐지 결과와 실제 감사 결과 대조 | PoC 구현 완료, 검증 진행 중 |
| 리스크 검토 리드타임 | 70% 단축      | 기존 수기 검토 시간 대비 케이스 처리 시간 비교 | PoC 구현 완료, 검증 진행 중 |

### 핵심 기능 정의

#### 1. 자율적 규정 내재화 및 추론 (Reasoning)

* **지능형 지식 파싱(Contextual Ingestion):** 규정집을 RAG 문서로 인덱싱·청킹(계층/조항/슬라이딩 전략)하고, BM25/Dense/RRF/Rerank 기반 검색으로 비즈니스 가이드라인 수준의 문맥을 제공. `policy_rulebook_probe` 등으로 탐지 로직에 즉시 반영되는 **지식 기반 자율 활용**을 수행한다.

* **다단계 추론 체인(Chain-of-Thought):** 단순 키워드 매칭이 아니라, 아래 노드 흐름으로 "왜 이 전표가 위반인가?"에 대한 **사고 과정(Reasoning)**을 설계·기록한다. 각 노드의 reasoning 및 도구 결과는 SSE 이벤트와 최종 분석 결과에 반영된다.
  * `intake`(전표 입력 정규화·플래그 추출)
  * `planner`(케이스 유형별 조사 계획 수립·도구 선택)
  * `execute`(LangChain 도구 순차/병렬 실행·점수 산정)
  * `critic`(도구 결과 기반 과잉 주장·반례 검토)
  * `verify`(게이트 적용·HITL 필요 여부 판단)
  * `reporter`(근거 기반 최종 설명·요약 생성)
  * `finalizer`(상태·점수·이력 최종 확정)

#### 2. 근거 기반 자율 조치 제안 (Self-Directed Action)

* **근거 객체 자동 맵핑(Evidence Graphing):** AI가 위반 사례를 발견하면, 아래 전문 도구(tool)들로 관련 전표·규정 조항·휴일/예산/가맹점 정보를 취합하여 **증거 패키지**를 구성한다. 인간이 일일이 데이터를 찾을 필요를 줄이는 에이전틱 조치이다.
  * `document_evidence_probe`(전표 라인아이템과 문서 증거 수집)
  * `policy_rulebook_probe`(내부 규정집에서 관련 조항 조회)
  * `holiday_compliance_probe`(휴일/휴무/연차 사용 정황 검증)
  * `budget_risk_probe`(예산 초과 여부와 금액 지표 검증)
  * `merchant_risk_probe`(거래처·가맹점 업종 코드(MCC) 기반 위험도 검증)

* **지능형 조치 가이드(Prescriptive Guidance):** 탐지에서 끝내지 않고, reporter(근거 기반 최종 설명·요약)·finalizer(상태·점수·이력 최종 확정)에서 근거와 점수(score_breakdown)를 바탕으로 **조치 시나리오를 자율적으로 요약**하여 인간의 의사결정을 가이드한다. Streamlit AI 워크스페이스에서 실시간 사고 흐름·결과·근거를 확인할 수 있다.

#### 3. 가드레일 기반의 인간-AI 협업 (HITL Logic)

* **구조화된 HITL:** 핵심 필드 누락, 증거 부족, 규정 해석 모호, specialist 도구 결과 충돌 시 `hitl_request`(reasons, questions, handoff)를 생성하고 **hitl_pause**에서 interrupt. 사용자는 UI/API로 응답 제출 후 **같은 run**에서 재평가(resume) 및 최종 확정까지 이어간다.

* **검토 반영 및 재분석:** 사람이 AI의 판단을 수정·승인하면, 해당 응답이 상태에 반영되어 **hitl_validate → reporter(최종 설명·요약) → finalizer(최종 확정)**만 재실행되며, 최종 설명과 판단요약이 갱신된다. (체크포인트는 `CHECKPOINTER_BACKEND=postgres` 사용 시 DB에 저장되어 재시작 후에도 HITL 직후부터 재개 가능.)

### 기대 효과

* **업무 지능화:** AI가 선별한 핵심 케이스에만 집중하여 재무팀의 업무 패러다임을 '단순 검토'에서 '고도화된 리스크 관리'로 전환.  
* **부정 방지 강화:** 인간이 놓치기 쉬운 변칙적 지출 패턴을 전수 조사(스크리닝 + 도구 기반 심층 분석)하여 재무 사고 원천 차단.  
* **거버넌스 확보:** 모든 AI 판단 근거·이벤트가 DB/런타임에 기록되며, run 이벤트 API·SSE 스트림·Streamlit 에이전트 스튜디오를 통해 향후 감사 시 객관적인 증빙 자료로 활용 가능.

### 범위 및 제약사항

* **In Scope**
  * 규정 기반 리스크 탐지 (스크리닝 + LangGraph 심층 분석, 도구 기반 증거 수집)
  * AI 추론 근거 제시 UI (Streamlit AI 워크스페이스, 실시간 이벤트·타임라인, HITL 응답)
  * 시연용 데이터 생성기 (시나리오별 전표 생성/삭제, BE 규칙과 정합성 유지)
  * 가드레일 기반 승인 워크플로우 (HITL interrupt/resume, 구조화된 hitl_request)
  * LangGraph 에이전트 오케스트레이션, SSE 실시간 이벤트 스트리밍, RAG 규정집 통합(청킹 실험실·품질 리포트), FastAPI + Streamlit 단일 스택 PoC

* **Out of Scope**
  * 완전 무인 자동 실행(무승인) 운영은 초기 제외(단계적 확대)

* **제약사항**
  * SAP 인터페이스(API) 응답 속도 및 보안 정책에 따른 데이터 접근 권한 제한 우려
  * SAP 연동은 운영 환경에 따라 배치(S3/적재) 우선 적용 후, 필요 시 실시간(JCo/API) 확장 방식으로 단계 적용
  * PoC는 인증 미적용, tenant_id/user_id 고정(1), 메모리 기반 런타임 일부 사용으로 시연/검증 목적에 한정
