## **주요 문제 해결 및 기술 리서치 (테스트 단계)**

테스트 단계에서는 단순 정확도보다 **에이전트 동작의 신뢰성(정합성/재현성/설명 가능성)**에 중점을 두고 개선했다.

| **이슈 구분** | **문제 상황 및 원인** | **리서치 및 해결 과정 (Reference & Solution)** |
| --- | --- | --- |
| **정합성** | 케이스 맥락과 맞지 않는 규정/질문이 생성됨 | `policy_case_alignment` 필터와 verify 후처리 강화로 케이스-조항/질문 정렬 |
| **HITL UX** | 재검토 시 이전 질문/답변 재확인이 어려움 | run 이력에서 이전 제출값 fallback 로드, 팝업에 재표시 로직 추가 |
| **스크리닝 신뢰성** | 생성 시 스크리닝과 분석 시 분류가 중복/불일치될 수 있음 | `start_router` 분기로 사전 스크리닝 결과 재사용, 중복 분류 제거 |
| **멀티모달 추출 품질** | 거래일시/합계금액 하이라이트가 케이스별로 흔들림 | 라벨-값 매핑 규칙 개선, 케이스별 후처리 룰 분리(분리형/결합형 날짜시간 대응) |
| **운영 가시성** | "AI가 실제로 동작했는지" 설명 근거 부족 | SSE 이벤트와 run 타임라인을 발표 증빙으로 표준화 |

## 1. LLM 답변 품질 평가 및 개선

| 항목 | 내용 |
| --- | --- |
| 평가 대상 기능 | 규정 인용 정확성, 검토 질문 맥락 일치도, 최종 판단 설명 일관성 |
| 평가 방식 | 케이스별 E2E 실행 후 `policy_refs`/`review_questions`/`final_result` 수동 검증 + 회귀 테스트 |
| 평가 결과 | 정상/휴일/한도초과 기준으로 핵심 흐름 동작 확인. 일부 케이스에서 질문 맥락 이탈 재현됨 |
| 개선 조치 | 케이스-조항 필터, required_inputs 누락 필드 검증, NORMAL_BASELINE 예외 처리 강화 |
| 개선 후 결과 | 맥락 이탈 빈도 감소, 불필요 HITL/질문 감소. 단, 모델 변동성 구간은 지속 모니터링 필요 |
| 잔여 과제 | 자동 평가셋(정답 라벨) 구축 및 정량 대시보드화 |

## 2. 성능 및 비용 최적화

| 항목 | 내용 |
| --- | --- |
| 기존 병목 | 동일 run 내 중복 분류/불필요 도구 호출로 지연 발생 |
| 개선 전략 | 라우팅 분기 명확화, 케이스별 도구 선택 최소화, 불필요 HITL 감소 |
| 적용 기술 | `start_router`, tool plan 최적화, required_inputs 필터링, UI 재렌더 최소화 일부 적용 |
| 개선 결과 | 체감 지연 감소 및 재검토 루프 단축. 정식 비용 지표는 운영 로그 집계 후 확정 예정 |

주의: 현재는 PoC 단계로, "절대 성능 수치"보다 "구조적 병목 제거"를 우선 적용했다.

## 3. 예외 처리 및 가드레일

| 항목 | 내용 |
| --- | --- |
| 차단 대상 | 근거 없는 확정 판정, 상충 증거 무시, 케이스 맥락 불일치 규정 채택 |
| 탐지 방식 | critic/verify 단계에서 근거 충실도 및 불일치 분류(taxonomy) 점검 |
| 대응 로직 | fail-closed: 근거 부족/충돌 시 자동 통과 대신 HITL 승격 |
| 테스트 결과 | 고위험 케이스에서 "검토 필요" 우선 전략 유지. 정상 케이스 과검출 완화 로직 동작 확인 |

## 4. 기타 문제 해결 사례

- **베타 생성 UI 정합성 개선**
  - `user_reason` 제거, `적요(bktxt)` 비필수화
  - 정상 비교군 선택 시 증빙 업로드 비활성화
  - 생성건수(다건 생성) 지원
- **HITL 재진입 UX 보완**
  - 재오픈 시 기존 질문/답변 표시
  - 질문 미노출 케이스에 required_inputs 기반 fallback 질문 제공
- **정책 점수 해석 보정**
  - 정책/근거 점수 문구가 의미와 반대로 표현되지 않도록 설명 로직 점검 및 수정

---

## 5. 테스트 체크리스트 (발표용)

1. 생성 시점 스크리닝 결과가 목록/상세/분석 run에서 일관되는가
2. 분석 실행 시 노드/도구 이벤트가 SSE로 관측되는가
3. 규정 조항 채택과 질문이 케이스 맥락과 일치하는가
4. HITL 응답 후 동일 run 재개가 되는가
5. 정상 비교군에서 불필요한 HITL이 억제되는가

---

## 6. 고도화 로드맵

1. 자동 평가셋 구축: 케이스별 정답 조항/질문/판정 라벨 확보
2. 정량 KPI 대시보드: precision/recall, HITL 발생률, resume 성공률 자동 집계
3. 멀티모달 안정화: OCR 블록 신뢰도 기반 라벨-값 매칭 재설계
4. 운영 안정화: 체크포인터/이력 영속화 기본값 강화 및 장애 복구 시나리오 추가

정리: 테스트 및 고도화의 핵심은 모델 교체 자체가 아니라, **워크플로우 제어 + 근거 정합성 + HITL 운영성**을 안정적으로 끌어올리는 것이다.
