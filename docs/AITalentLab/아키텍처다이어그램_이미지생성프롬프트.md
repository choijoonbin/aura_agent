## 한국어 요약 (생성 도구에 따라 활용)

**배치 구조 (위→아래):**
1. 사용자(Streamlit UI) → FastAPI 서버
2. LangGraph Agent Engine (핵심, 8개 노드 흐름도)
   - start_router → screener → planner → execute → critic → verify → reporter → finalizer
   - verify에서 HITL 노드로 조건 분기 (점선 복귀 화살표)
3. 도구 레이어: RAG Pipeline / Vision LLM / Azure OpenAI (3개 병렬)
4. 데이터: PostgreSQL(pgvector + 체크포인터) / 정책 문서
5. 우측 출력: 최종 보고서 / HITL 결과

**컬러 코드:**
- 네이비 `#1A365D`: 서버/인프라
- 틸 `#4ECDC4`: AI 에이전트 노드
- 밝은 회색 `#F5F5F5`: 데이터 저장소
