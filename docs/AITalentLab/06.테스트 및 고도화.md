## 테스트 전략 및 품질 관리

PoC의 신뢰성을 보장하기 위해 **자동화 테스트 + LLM-as-a-Judge 평가 + 회귀 방지**의 3계층 전략을 수립합니다.

### 0-2. 테스트 데이터(대표 시나리오 3개 + 엣지 케이스)

- **대표 시나리오(2~3개)**
  - **S1: HOLIDAY_USAGE**: 휴일 + 근태 충돌 + MCC(고위험) → verify에서 HITL 요청 → 담당자 승인 후 `COMPLETED_AFTER_HITL`
  - **S2: LIMIT_EXCEED**: 예산 초과 플래그/금액 구간 승인 기준 → 규정 인용 + 스코어링 일관성 검증
  - **S3: NORMAL_BASELINE**: 위험 신호 미약 → 자동 완료(불필요 HITL/과탐 방지)
- **엣지 케이스**
  - **E1: 문서(라인아이템) 누락**: `document.items` empty → HITL 강제(required_inputs에 `document_items`)
  - **E2: OCR/멀티모달 실패 또는 저신뢰**: `_VISUAL_MIN_CONFIDENCE=0.6` 미만 엔티티는 비교 생략, 금액 불일치(>5%)는 차단
  - **E3: 규정 근거 연결 불충분**: `coverage_ratio < 1.0`이면 자동 확정 중단 사유로 적재 → HITL 유도

## 6. 가드레일/HITL 기준을 “규칙”으로 문서화 (정량/조건)

### 6-1. 자동 확정(READY) vs HITL(HITL_REQUIRED) 규칙

아래 조건 중 **하나라도 만족하면 자동 확정을 중단하고 HITL로 승격**합니다.

- **검증 게이트 정책 상태 존재**: `verification_summary.gate_policy`가 존재(예: hold/caution/regenerate_citations 등)
- **근거 연결률 미달**: `coverage_ratio < 1.0`이면 “자동 확정 기준 미달”로 기록되고 HITL에 반영
- **critic 보류 권고**: `critique.recommend_hold == true`
- **규정 위반 유형 + 중간 이상 심각도**
  - `case_type in {HOLIDAY_USAGE, LIMIT_EXCEED, PRIVATE_USE_RISK, UNUSUAL_PATTERN}`
  - AND `severity in {MEDIUM, HIGH, CRITICAL}`
- **입력/증빙 누락**
  - `body_evidence.dataQuality.missingFields` 존재
  - OR `document.items` 비어 있음(라인아이템 없음)
- **규정 기반 동적 HITL**
  - 규정 본문에서 파생된 `required_inputs` 또는 `review_questions`가 존재할 때(단, NORMAL_BASELINE은 예외적으로 skip 가능)

UI 과밀 방지 규칙:

- **질문 최대 2개**: `MAX_HITL_QUESTIONS=2`
- **필수 입력 최대 2개**: `MAX_HITL_INPUTS=2`

### 6-2. 실패 시 처리(예시 2개)

- **케이스 A: 근거 없는 주장(no_citation) 또는 증거 모순(contradictory_evidence)**
  - 처리: **fail-closed** → 자동 확정 금지 → HITL 요청 이벤트 기록 → 담당자 판단/보완 후 재개
- **케이스 B: 멀티모달 증빙 불일치(금액)**
  - 규칙: 이미지 추출 금액 vs 입력 금액 편차가 **5% 초과**면 `contradictory_evidence`로 분류(차단)
  - 처리: HITL로 에스컬레이션 + 담당자에게 “입력/영수증 중 무엇이 정답인지” 확인 요청

### 6-3. 스크리닝 가드레일(승격/보정) 규칙

- **LLM 신뢰도 경계값**: `llm_confidence < 0.75`면 결정론 결과 우선(하이브리드 정렬)
- **rule-LLM 불일치**: 불일치 시 deep lane 승격(추가 도구/검증으로 과탐/미탐 완화)
- **NORMAL_BASELINE 보호**: 위험 신호가 충분치 않으면 정상군을 과도 승격하지 않도록 보호 규칙 적용

## 8. 예외 처리 및 가드레일(요약 표)


| 항목              | 내용                                                                                                                                                                                                                                                          |
| --------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **가드레일 대상**     | LLM 스크리닝 환각(시그널 없는 케이스 유형 오분류), 근거 없는 주장(no_citation), 모순 판단(contradictory_evidence), 멀티모달 증빙 불일치(>5%), LLM judge 연속 오류                                                                                                                                     |
| **탐지 방식**       | `screener.py`의 결정론적 가드레일(`_align_hybrid_case_type()`), `langgraph_nodes_review.py`의 blocking taxonomy 기반 fail-closed, `langgraph_reasoning.py`의 `check_reasoning_consistency()`, LLM judge circuit breaker(`llm_judge_circuit_window`, `failure_threshold`) |
| **대응 로직**       | 스크리닝 환각 → 결정론적 결과로 자동 보정 + `hybrid_case_align_reason` 기록. 근거 없는 주장 → HITL 강제 승격. LLM judge 연속 실패 → circuit breaker 발동, rule_score로 안전하게 fallback.                                                                                                           |
| **Fallback 체계** | 모든 LLM 호출에 구조화 출력 파싱 실패 대비 Pydantic 기본값 fallback. `fallback_used=true`, `fallback_reason` 필드로 진단 로그 기록.                                                                                                                                                     |
