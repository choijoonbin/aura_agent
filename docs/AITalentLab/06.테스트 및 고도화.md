## **주요 문제 해결 및 기술 리서치 (테스트 단계)**

테스트 과정에서 발견된 엣지 케이스(Edge Case)와 성능 병목을 해결하기 위해 리서치하고 적용한 방법을 기록합니다.

| **이슈 구분**     | **문제 상황 및 원인**                                                             | **리서치 및 해결 과정 (Reference & Solution)**                                                                                                                                                                              |
| ------------- | -------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **품질/환각**     | 판단요약에서 정책점수 의미가 반대로 해석되거나, 근거 연결이 약한 주장(unsupported claim)이 자동 확정될 여지가 있었음 | •리서치: 점수 의미 일관성 보정 및 근거 중심 검증 강화 • 적용: `agent/langgraph_scoring.py`에 summary 문구 sanitizer 적용, `agent/langgraph_nodes_review.py`에서 blocking taxonomy 기반 fail-closed(HITL 강제) 적용                                      |
| **속도/지연**     | 이미지 OCR 처리 및 LLM 추론 연쇄 호출로 인한 전체 응답 시간 지연                                  | •**리서치:** LangGraph의 비동기 노드 실행 및 중복 OCR 처리 방지 기법 • **적용:** `persistence_service.py`를 통한 OCR 결과 캐싱 및 LangGraph 내 `async` 노드 처리를 통해 중복 작업을 제거하여 처리 속도 개선                                                              |
| **보안/가드레일**   | 증빙 데이터 분석 중 민감 정보(계좌번호 등) 노출 우려                                            | • **리서치:** 전처리 단계에서의 PII(개인정보) 탐지 및 마스킹 가드레일 조사 • **적용:** `screener.py` 모듈 내에 입력 전처리 필터를 추가하여 민감 정보 포함 시 리뷰 노드로 강제 라우팅하여 안전성 확보                                                                                     |
| **RAG/검색 품질** | 사내 규정집의 복잡한 계층 구조로 인해 단순 시맨틱 검색 시 관련 규정 누락 및 환각 발생                         | • **리서치**: Hybrid Search(Keyword + Semantic) 및 RRF(Reciprocal Rank Fusion) 기법 리서치 • **적용**: graph_db_[service.py](http://service.py)를 통해 규정 간 연관 관계를 그래프화하고, RRF 알고리즘을 적용하여 키워드와 문맥 검색 결과를 결합함으로써 검색 재현율(Recall) 향상 |

• **적용:** `screener.py` 모듈 내에 입력 전처리 필터를 추가하여 민감 정보 포함 시 리뷰 노드로 강제 라우팅하여 안전성 확보

## 4. 기타 문제 해결 사례

**이미지 스케일링 보정**: UI(`demo_new.py`)에서 하이라이트가 어긋나는 문제는 Streamlit의 캔버스 렌더링 크기와 원본 이미지의 픽셀 좌표계 차이에서 기인했습니다. 이를 해결하기 위해 OCR 좌표를 0~1 사이의 정규화된 값(Normalized Coordinates)으로 변환하여 전달하고, 프론트엔드 노드에서 표시 크기에 맞춰 재계산하는 로직을 적용했습니다.

데이터 영속성 관리: runtime_persistence_[service.py](http://service.py)를 통해 에이전트의 중간 추론 과정(Thinking Process)을 DB에 기록함으로써, 추후 하이라이트 실패 원인을 사후 분석(Post-mortem)할 수 있는 가시성을 확보했습니다.
