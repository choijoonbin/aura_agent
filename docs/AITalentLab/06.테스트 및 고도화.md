## 테스트 전략 및 품질 관리

PoC의 신뢰성을 보장하기 위해 **자동화 테스트 + LLM-as-a-Judge 평가 + 회귀 방지**의 3계층 전략을 수립합니다.

---

| 항목 | 내용 |
|------|------|

| **평가 방식** | 규정집(사내 경비 지출 관리) 기반 골드 템플릿(`docs/Edu/retrieval_gold_template.jsonl`) 구축 후 `scripts/evaluate_retrieval_quality.py`로 LLM-as-a-Judge(GPT-4o) 자동 평가 |

| **개선 조치** | `policy_ref_normalizer.py` 도입으로 규정 참조 형식 통일(환각 방지). 시스템 프롬프트에 "입력에 없는 사실을 만들지 마라" 명시. BM25+Dense+RRF 하이브리드 검색으로 recall 향상. 조문(article) 단위 중복 제거로 관련도 낮은 청크 상위 노출 억제. |
| **개선 후 결과** | Faithfulness 89%, Answer Relevance 92% |
| **평가 도구** | `services/retrieval_quality.py`, `services/citation_metrics.py`, `scripts/evaluate_retrieval_quality.py` |

## 3. 주요 문제 해결 사례 (테스트 단계)

테스트 과정에서 발견된 엣지 케이스(Edge Case)와 성능 병목을 해결하기 위해 리서치하고 적용한 방법을 기록합니다.

| **이슈 구분** | **문제 상황 및 원인** | **리서치 및 해결 과정 (Reference & Solution)** |
| ------------- | ---------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **품질/환각** | 판단요약에서 정책 점수 의미가 반대로 해석되거나, 근거 연결이 약한 주장(unsupported claim)이 자동 확정될 여지가 있었음 | **리서치:** 점수 의미 일관성 보정 및 근거 중심 검증 강화 패턴 조사. **적용:** `langgraph_scoring.py`에 summary 문구 sanitizer 적용. `langgraph_nodes_review.py`에서 blocking taxonomy(`no_citation`, `contradictory_evidence`) 기반 fail-closed(HITL 강제) 적용. `check_reasoning_consistency()`로 reasoning↔verdict 일관성 자동 검증. |
| **속도/지연** | LLM 추론 연쇄 호출(planner→execute→critic→verify→reporter→finalizer) 및 이미지 OCR 처리로 인한 전체 응답 시간 지연 | **리서치:** LangGraph 비동기 노드 실행 및 중복 처리 방지 기법 조사. **적용:** LangGraph 내 `async` 노드 처리로 I/O 대기 최소화. `evidence_extraction.py`에서 SHA-256 해시 기반 OCR 결과 캐싱으로 동일 영수증 재처리 방지. `ENABLE_PARALLEL_TOOL_EXECUTION=true`로 독립 도구 병렬 실행. |
| **RAG 검색 품질** | 사내 규정집의 복잡한 계층 구조(장·절·조·항·호)로 인해 단순 시맨틱 검색 시 관련 규정 누락 및 조문 경계 붕괴 | **리서치:** Hybrid Search(BM25+Dense), RRF(Reciprocal Rank Fusion), cross-encoder rerank 기법 리서치. **적용:** `policy_service.search_policy_chunks()`에서 BM25·Dense 후 RRF 결합, LLM fallback rerank(`ENABLE_LLM_RERANK_FALLBACK=true`). 조문(article) 단위 중복 제거 및 `adoption_reason` 기록. 검색 재현율(Recall) 향상 확인. |
| **HITL 재개 불안정** | `CHECKPOINTER_BACKEND=memory` 환경에서 멀티 워커 배포 시 체크포인트를 찾지 못해 screener부터 전체 재실행되는 문제 | **리서치:** LangGraph checkpointing 공식 문서 및 `langgraph-checkpoint-postgres` 라이브러리 검토. **적용:** `CHECKPOINTER_BACKEND=postgres` 설정 시 DB에 체크포인트 저장하여 재시작 후에도 HITL 직후부터 정확하게 재개. PoC 기본은 memory(단일 프로세스 보장). |
| **스크리닝 과탐** | 정상 전표(NORMAL_BASELINE)를 LLM이 UNUSUAL_PATTERN 등으로 과도하게 위험 분류하는 케이스 발생 | **리서치:** 정상군 과탐 억제(false positive 방지) 패턴 조사. **적용:** `_align_hybrid_case_type()`에 `normal_baseline_protected_low_score`(점수 < 20), `normal_baseline_protected_pattern_escalation`(점수 < 30 + UNUSUAL_PATTERN 제안) 가드레일 추가. LLM confidence < 0.75이면 결정론적 결과 채택. |

## 4. 성능 및 최적화

| 항목 | 내용 |
|------|------|
| **기존 병목** | 매 분석 run마다 모든 LLM 노드(planner→execute→critic→verify→reporter→finalizer) 순차 호출로 전체 응답 시간 과다 |
| **개선 전략** | 독립 도구 병렬 실행 + 비동기 노드 + OCR 해시 캐싱 |
| **적용 기술** | `ENABLE_PARALLEL_TOOL_EXECUTION=true`: holiday_compliance_probe·budget_risk_probe·document_evidence_probe 병렬 그룹 실행. `async` LangGraph 노드로 I/O 대기 최소화. `evidence_extraction.py` SHA-256 해시 캐싱으로 동일 이미지 재처리 방지. |
| **개선 결과** | 독립 도구 병렬화로 execute 단계 처리 시간 단축. 동일 영수증 재분석 시 OCR 처리 생략. |

## 5. 예외 처리 및 가드레일

| 항목 | 내용 |
|------|------|
| **가드레일 대상** | LLM 스크리닝 환각(시그널 없는 케이스 유형 오분류), 근거 없는 주장(no_citation), 모순 판단(contradictory_evidence), LLM judge 연속 오류 |
| **탐지 방식** | `screener.py`의 결정론적 가드레일(`_align_hybrid_case_type()`), `langgraph_nodes_review.py`의 blocking taxonomy 기반 fail-closed, `langgraph_reasoning.py`의 `check_reasoning_consistency()`, LLM judge circuit breaker(`llm_judge_circuit_window`, `failure_threshold`) |
| **대응 로직** | 스크리닝 환각 → 결정론적 결과로 자동 보정 + `hybrid_case_align_reason` 기록. 근거 없는 주장 → HITL 강제 승격. LLM judge 연속 실패 → circuit breaker 발동, rule_score로 안전하게 fallback. |
| **Fallback 체계** | 모든 LLM 호출에 구조화 출력 파싱 실패 대비 Pydantic 기본값 fallback. `fallback_used=true`, `fallback_reason` 필드로 진단 로그 기록. |

## 6. 고도화 내용

**Phase A → Phase B → Phase C 점진적 고도화**

| 고도화 단계 | 주요 내용 | 적용 결과 |
|-------------|-----------|-----------|
| **Phase A: 기본 에이전트** | LangGraph 기본 노드(intake→planner→execute→verify→reporter→finalizer), 결정론적 스크리닝, 기본 RAG(Dense 단일 검색) | 기본 분류·판단 흐름 구현 |
| **Phase B: 구조화 출력 + HITL** | Pydantic 구조화 출력(PlannerOutput/CriticOutput/VerifierOutput/ReporterOutput), critic 노드 추가, HITL interrupt/resume, SSE 이벤트 스키마 고도화 | 설명 가능성·신뢰성 대폭 향상 |
| **Phase C: 하이브리드 고도화** | 하이브리드 스크리닝(LLM+가드레일), 하이브리드 스코어링(rule+LLM judge), BM25+Dense+RRF+Rerank Hybrid Search, 멀티모달 OCR, start_router 분기, Langfuse 통합, Neo4j Graph DB(선택), PostgreSQL 체크포인터 | 정확도 향상, 운영 견고성 확보, 관찰 가능성 강화 |
