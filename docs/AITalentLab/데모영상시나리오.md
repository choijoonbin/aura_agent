# AuraAgent 데모 영상 시나리오

> **대상:** AI Master Project 최종 발표 제출용 데모 영상
> **총 길이:** 약 5분 (로딩 구간 2배속 편집 적용)
> **핵심 어필:** 자율형 Agentic AI — Planning · Deep Reasoning · Self-Correction · HITL 협업
> **시연 구성:** HOLIDAY_USAGE(고위험) 4단계 연속 흐름 + NORMAL_BASELINE(정상) 대비
>
> | 단계 | 메뉴 | 내용 |
> |------|------|------|
> | ① OCR 증빙 검증 | 시연데이터 생성 | 영수증 업로드 → Vision LLM + PaddleOCR 2단계 분析 |
> | ② 하이브리드 스크리닝 | 시연데이터 생성 | 테스트 데이터 생성 → LLM·규칙 이중 판정 → CRITICAL 분류 |
> | ③ Agentic AI 심층 분析 | AI 워크스페이스 | 분析 시작 → Planning → Execute(4도구 병렬+RAG) → Self-Correction |
> | ④ HITL 협업 | AI 워크스페이스 | FAIL-CLOSED 게이트 → 담당자 검토 팝업 → 이어하기 |
> | ⑤ 정상 케이스 대비 | AI 워크스페이스 | NORMAL_BASELINE — HITL 없이 자동 완료 (대비 강조) |

---

> ⚠️ OCR·Vision LLM 로그는 **Streamlit 터미널**에 출력됩니다

```
[vllm:analyze] 증빙 이미지 분석 시작 | model=gpt-4o timeout=45.0s
[vllm:ocr] PaddleOCR 실행 시작
[vllm:ocr] PaddleOCR 3.4 완료 | blocks=32 avg_conf=0.94 img=2816x1536
[vllm:ocr] 완료: 32개 텍스트 블록 추출
[vllm:llm] OCR→LLM 필드 매핑 요청 | model=gpt-4o ocr_blocks=32
[vllm:match] LLM 매핑 결과 | merchant(k=0,v=0) | date(k=4,v=5) | time(k=6,v=7) | amount(k=27,v=28)
[vllm:result] 추출완료 | condition=clear avg_conf=0.94 entities=4개
[vllm:result] merchant_name="가온식당 강남점"(conf=0.99) | date_occurrence="2026-03-14"(conf=1.00) | time_occurrence="23:42"(conf=1.00) | amount_total="68000"(conf=1.00)
[vllm:analyze] PaddleOCR+LLM 경로 완료 | entities=4개
```

**멘트:**

### ▶ 1:00 ~ 1:30 | 전표 생성 + 하이브리드 스크리닝

**화면:** 「테스트 데이터 생성」 클릭 → Streamlit 터미널 연속 확인

**[행동] 「테스트 데이터 생성」 클릭**

**멘트:**
> "이제 테스트 데이터 생성을 누르겠습니다. 결제 시각은 토요일 23시 42분, 업종은 고위험 업종코드 5813, 근태는 LEAVE — 연차입니다.
> 전표가 생성되는 순간, AI가 자동으로 하이브리드 스크리닝을 시작합니다.
> 실제 운영에서는 전표를 인터페이스 받으면 자동으로 하이브리드 스크리닝 단계가 동작한다고 보시면 됩니다."

**[Streamlit 터미널에서 강조할 로그 — 마우스로 해당 줄 가리키며 설명]**

```
[screener:llm] LLM 분류 요청 | model=gpt-4o-mini response_format={'type': 'json_object'}
[screener:llm] LLM 제안 | model=gpt-4o-mini case=HOLIDAY_USAGE conf=0.9 reason=휴일에 경비가 발생하여 분류되었습니다.
[screener:llm] LLM 분류 결과 | llm_case=HOLIDAY_USAGE(conf=0.90) vs rule_case=HOLIDAY_USAGE(score=95) min_conf=0.75
[screener:guardrail] 최종결정 | LLM 채택 → case=HOLIDAY_USAGE | reason=llm_rule_agree
[screener:result] 스크리닝 완료 | case=HOLIDAY_USAGE score=95 severity=CRITICAL llm=HOLIDAY_USAGE(conf=0.90) align=llm_rule_agree
```

> **각 숫자 산출 근거 (발표자 참고용)**
>
> | 항목 | 값 | 설명 |
> |------|-----|------|
> | `conf=0.90` | LLM 출력값 | gpt-4o-mini가 전표 컨텍스트 분석 후 직접 반환한 확률 (0~1 스케일) |
> | `score=95` | 규칙 엔진 합산 | 토요일(+35) + LEAVE 근태(+20) + 23시 심야(+10) + MCC 5813 고위험(+30) = **95** |
> | `severity=CRITICAL` | 점수 기준 분류 | score ≥ 70 → CRITICAL |
> | `min_conf=0.75` | 설정값 | LLM 채택 최소 신뢰도 임계값 (0.90 > 0.75 → 채택) |

**멘트:**
> "두 줄만 보시면 됩니다.
>
> **두 번째 줄** — LLM이 전표 내용을 보고 직접 판단했습니다. 토요일, 23시 42분, 연차 직원, 고위험 업종. 이 맥락을 종합해서 confidence=0.90 — 90% 확신으로 '휴일사용의심'을 반환했습니다.
>
> **세 번째 줄이 핵심입니다.** 규칙 엔진이 별도로 위험 요소를 점수화했습니다. 토요일 +35, 연차 +20, 심야 +10, 고위험업종 +30 — 합산 score=95. LLM 판단과 규칙 점수, **둘 다 같은 결론**입니다(`llm_rule_agree`).
>
> 가드레일이 이 합의를 확인하고 최종 확정합니다. severity=CRITICAL. 단순 LLM이었다면 맥락 이해 오류로 통과시킬 수 있는 케이스도, 규칙 엔진이 수치로 교차 검증하기 때문에 오분류를 막습니다."

**[UI 화면으로 전환]**

**멘트:**
> "워크스페이스 전표 목록으로 돌아오면, 이미 **위험도 높음** 배지가 붙어 있는 것을 확인할 수 있습니다.
> 이제 이 전표를 선택하고 심층 분석을 실행해보겠습니다."
