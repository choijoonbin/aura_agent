# Scoring Rubric v1

## 평가 축 (3개)

| 축 | 범위 | 설명 |
|---|---|---|
| policy | 0~100 | 규정 위반 정황의 강도 (휴일/심야/근태충돌/업종/한도) |
| evidence | 0~100 | 규정 조항·전표 라인·도구 결과의 증거 충실도 |
| fidelity (grounding) | 0~100 | 결론이 실제 증거에 의해 직접 지지되는 정도 |

---

## 핵심 가산 기준 — policy 점수

| 조건 | 기준 가산점 | 비고 |
|---|---|---|
| 휴일/주말 사용 (`isHoliday=true`) | +35 | 가장 높은 단일 신호 |
| 근태 충돌 (`hrStatus=LEAVE/OFF/VACATION`) | +20 | 휴일과 중복 시 복합 승수 적용 |
| 심야 시간대 (22시~06시) | +10 | 23시~06시 식대 경고(제23조 ③-1) |
| 예산 초과 (`budgetExceeded=true`) | +15 | 제40조/제19조 한도 규정 |
| 가맹점 고위험 (`merchantRisk=HIGH`) | +20 | 도구 확인 결과 반영 |
| 가맹점 중위험 (`merchantRisk=MEDIUM`) | +10 | 도구 확인 결과 반영 |
| 가맹점 저위험 (`merchantRisk=LOW`) | +3 | |
| 휴일+근태 중복 (HIGH) | 추가 +10 | `isHoliday=true AND hrStatus∈{LEAVE,OFF,VACATION}` |
| 휴일만 확인 (MEDIUM) | 추가 +5 | `isHoliday=true` 단독 |

## 복합 위험 승수

고위험 신호(isHoliday, hrStatus충돌, isNight, budgetExceeded, merchantRisk=HIGH)가 동시에 감지될 때 정책 점수에 곱해진다.

| 동시 감지 고위험 수 | 승수 |
|---|---|
| 4개 이상 | ×1.5 (최대) |
| 3개 | ×1.3 |
| 2개 | ×1.15 |
| 1개 이하 | ×1.0 (미적용) |

## 패널티 조건

| 조건 | 효과 |
|---|---|
| 도구 성공률 < 50% | evidence_score에서 비례 감점 (최대 -20점) |
| 고위험 3개 이상 + 증거 완결성 < 40% | 최종 점수 추가 감점 |

## 의사결정 게이트 기준

| 최종 점수 (final_score) | severity | 기준 동작 |
|---|---|---|
| 75점 이상 | CRITICAL | HITL 검토 요청 |
| 55~74점 | HIGH | caution 상태, 추가 검토 권고 |
| 35~54점 | MEDIUM | 부분 검토 |
| 34점 이하 | LOW | 자동 확정 가능 |

## LLM Judge 평가 원칙

1. **규칙 엔진 결과 존중**: `rule_violation_summary`에 기재된 위반 사항은 반드시 policy_score에 반영한다. 규칙 엔진이 탐지한 항목을 LLM이 무시하거나 과소평가하면 안 된다.
2. **과대평가 금지**: 증거가 없는 위반을 임의로 추가하지 않는다.
3. **일관성**: 동일한 입력에 대해 동일한 점수 범위(±5점 이내)를 유지한다.
4. **fidelity 산정**: 결론이 전표·규정·도구 결과에 의해 직접 뒷받침되는 정도. 증거 없이 추론만으로 도달한 결론은 낮게 평가한다.

## 출력 원칙

- `summary_reason`: 사용자가 이해할 수 있는 한국어 2~3문장 요약. 전문 용어·코드 최소화.
- `internal_reason`: 각 점수 항목의 가산/감점 근거를 구체적으로 기술. 조문 번호 포함 권장.
- `overall_score`: `policy_score × 동적가중치 + evidence_score × 동적가중치` 기반 산출. 단순 평균 금지.
