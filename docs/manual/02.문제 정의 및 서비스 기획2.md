### 프로젝트 명

Aura Agentic AI 기반 실시간 재무 리스크 탐지 및 지능형 자율 워크벤치

(실제 구현: **Aura Agent AI PoC** — FastAPI + Streamlit + LangGraph 단일 코드베이스)

---

### 배경 및 문제 정의

* **현재 상황 (As-Is):** 수기 중심의 재무 모니터링 및 정적 규칙 기반(Rule-based) 사후 통제 시스템 운영

* **Pain Point:** 방대한 SAP 전표 검토와 증빙 확인으로 인한 리드타임 증가, 담당자 숙련도 의존에 따른 품질 편차

  * **비용:** 단순 반복 검토(Open Items, 증빙 대조, 규정 대입)로 인적 자원 소모
  * **품질:** 케이스 분류/근거 수집/최종 승인 판단이 분절되어 누락 및 오판 리스크 상존

* **기회 요인:** 에이전트형 AI가 전표를 자율 분류하고, 규정집(RAG) 기반 근거를 제시하며, HITL(Human-in-the-Loop)로 검증 가능한 승인 흐름을 구성할 수 있음. 본 PoC는 **LangGraph 기반 자율형 에이전트**가 스크리닝→심층분석→HITL→최종확정까지 한 흐름으로 수행하고, 구조화 이벤트(SSE)와 이력 로그로 설명 가능한 감사 결과를 제공한다.

---

### 목표 사용자 (Target User)

* **재무/회계 실무자:** 고위험 전표를 우선 선별해 검토 효율을 높이고 싶은 담당자
* **CFO/관리자:** 규정 근거와 점수 기반으로 승인/보류 의사결정을 빠르게 수행해야 하는 의사결정자
* **내부 감사/준법 팀:** AI 판단 과정, 근거 조항, HITL 응답 이력을 감사 증빙으로 확보해야 하는 사용자

---

### 프로젝트 목표

* **정성적 목표**

  * **목표 1 (자율 판단):** 전표 리스크 분류와 조치 판단을 에이전트가 수행하는 자율형 분석 환경 구현
    → **현재:** `start_router → (screener|intake) → planner → execute → critic → verify → hitl_pause/hitl_validate → reporter → finalizer` 워크플로우 운영. 스크리닝은 **LLM + 결정론 가드레일 하이브리드**로 분류.

  * **목표 2 (설명 가능한 AI):** "왜 위반인가"를 규정 조항과 증거로 설명하는 근거 중심 분석
    → **현재:** `policy_rulebook_probe`(규정), `document_evidence_probe`(전표), `holiday/budget/merchant` probe 결과를 결합해 adoption_reason, 점수 분해(score_breakdown), 판단요약 생성.

  * **목표 3 (신뢰 기반 승인):** AI 판단과 사람 승인을 결합한 하이브리드 거버넌스
    → **현재:** verify에서 HITL 필요 시 interrupt(`hitl_pause`) 후, 동일 run에서 응답 반영(`hitl_validate`) 및 결과 재확정. 검토 이력/응답은 run 단위로 조회 가능.

* **정량적 목표 (KPI)**

| **성과 지표** | **목표 수치** | **측정 방법** | **현재 수준** |
| --- | --- | --- | --- |
| 이상 전표 탐지 정확도 | 95% 이상 | AI 분류 결과 vs 검토자 최종 판정 비교 | PoC 구현 완료, 시나리오별 검증 진행 중 |
| 리스크 검토 리드타임 | 70% 단축 | 수기 검토 대비 케이스 처리 시간 비교 | PoC 구현 완료, 실측 집계 보강 중 |
| HITL 재처리 성공률 | 95% 이상 | HITL 제출 후 동일 run 재개 성공 비율 | 기능 구현 완료, 운영 로그 기반 검증 중 |

### 핵심 기능 정의

#### 1. 자율적 규정 내재화 및 추론 (Reasoning)

* **지능형 규정 검색(RAG):** 규정집을 청킹/임베딩해 PostgreSQL(pgvector)에 저장하고, BM25 + Dense + RRF + 재정렬로 관련 조항을 검색한다. 조항별 `adoption_reason`을 생성하고 케이스 정합성 필터(예: 휴일/한도초과 문맥 불일치 제거)를 적용한다.

* **다단계 추론 체인:** 단순 룰 매칭이 아닌 노드 기반 분석으로 근거를 축적한다.
  * `screener`: 케이스 유형 분류(LLM 제안 + 가드레일 보정)
  * `intake`: 입력 정규화/신호 추출
  * `planner`: 케이스별 조사 계획/도구 선택
  * `execute`: 도구 실행(순차/병렬)
  * `critic`: 반례/과잉 주장 점검
  * `verify`: 게이트/HITL 필요 여부 판정
  * `hitl_pause`, `hitl_validate`: 사람 검토 인터럽트 및 반영
  * `reporter`, `finalizer`: 최종 설명/상태 확정

#### 2. 근거 기반 자율 조치 제안 (Self-Directed Action)

* **증거 패키지 자동 수집:**
  * `holiday_compliance_probe`
  * `budget_risk_probe`
  * `merchant_risk_probe`
  * `document_evidence_probe`
  * `policy_rulebook_probe`
  * `legacy_aura_deep_audit`(옵션)

* **조치 가이드 생성:** 규정 근거, 점수, 검증 신호를 바탕으로 승인/보류/재검토 방향을 요약한다. 최종 결과와 근거는 워크스페이스에서 탭 형태로 확인 가능하다.

#### 3. 가드레일 기반 인간-AI 협업 (HITL Logic)

* **구조화된 HITL:** `reasons`, `review_questions`, `required_inputs`, `evidence_snapshot`를 포함한 `hitl_request` 생성.
* **재개 방식:** 검토 응답 제출 시 동일 run/thread에서 재개(resume)하여 결과를 갱신.
* **이력/재검토 지원:** HITL 팝업에서 이전 제출 이력(질문/답변) 재확인 및 재작성 가능.

### 기대 효과

* **업무 효율화:** 고위험 전표 우선 처리로 검토 생산성 향상
* **품질 표준화:** 담당자별 편차를 줄이고 근거 중심 판정 일관성 강화
* **감사 추적성 확보:** 이벤트 스트림, run 이력, HITL 응답을 통한 사후 감사 증빙 강화

### 범위 및 제약사항

* **In Scope**
  * 전표 스크리닝 + LangGraph 심층 분석 + HITL 재개까지의 E2E 흐름
  * 규정집 RAG 검색(하이브리드 검색/재정렬) 및 조항 근거 제시
  * Streamlit 워크스페이스(실시간 이벤트, 판단요약, HITL 팝업)
  * 시연 데이터 제어(legacy) + 시연데이터 생성(beta)
    * Beta: 이미지 분석(엔티티 추출/하이라이트), 정상비교군 업로드 비활성화, 생성건수(1~20) 다건 생성, 전체 삭제 연동

* **Out of Scope**
  * 완전 무인 승인(무-HITL) 운영 전환
  * SAP 운영계 실시간 양방향 연동의 상용 수준 안정화

* **제약사항**
  * PoC 특성상 인증/권한 모델은 단순화(기본 tenant/user 고정)
  * 런타임 일부는 메모리 기반이므로 배포 토폴로지에 따라 상태 복원 전략 점검 필요
  * 모델 응답 변동성으로 인한 질문/표현 편차가 존재해, 규칙 가드레일과 후처리 필터를 지속 보강해야 함
